<interfaces>
i18next interpolation syntax: `{{varName}}`. Translations MUST preserve every `{{...}}` placeholder EXACTLY — they are runtime substitution points. Changing `{{title}}` to `{{title }}` or localising the variable name breaks interpolation.

Natural-language standards for each locale:
- zh (Simplified Chinese): UI labels → short noun phrases. Button verbs → imperative verb + object. Status names follow standard software conventions ("待办" / "进行中" / "已完成" etc.).
- fr: formal tone (vous), no sentence-end period in UI buttons per common practice.
- de: noun capitalisation, formal tone (Sie).
- es: neutral Latin-American/European Spanish, "tú"-less where gendered ambiguity matters.
- it: formal register, matches the existing sidebar.* keys' register.
</interfaces>

|----------|-------------|
| Locale JSON → i18next runtime | Untrusted interpolation variables are escaped by React (since `interpolation.escapeValue: false` is offset by React's auto-escape in JSX) |
| Locale JSON → announcer `.textContent` | The @dnd-kit accessibility region sets `.textContent`, not `.innerHTML` — plan 23-04 §T-23-04-01 verified |
