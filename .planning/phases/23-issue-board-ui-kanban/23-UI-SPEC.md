## Spacing Scale

Use Tailwind v4 default scale (4-based multiples). Oxide CSS variables `--space-1..--space-10` exist for non-Tailwind contexts. **Prefer Tailwind utilities; fall back to CSS vars only for dynamic inline styles.**

| Token | Value | Usage on Board |
|-------|-------|----------------|
| `gap-1` / `--space-1` | 4px | Icon-to-label gap inside badges |
| `gap-2` / `--space-2` | 8px | Intra-card element spacing (title ↔ metadata row) |
| `gap-3` / `--space-3` | 12px | Card internal padding (`p-3`) |
| `gap-4` / `--space-4` | 16px | Column internal padding; card bottom margin between stacked cards |
| `gap-6` / `--space-6` | 24px | Page section padding (`p-6` on `IssuesBoardPage` root) |
| `gap-8` / `--space-8` | 32px | Gap between columns (`gap-4` if compact / `gap-6` if spacious — **pick `gap-4` for board density**) |

**Board layout exact values:**

| Element | Value | Rationale |
|---------|-------|-----------|
| Page root padding | `p-6` (24px all sides) | Matches existing `ChatHubPage` / `WorkbenchPage` |
| Board header → columns gap | `mb-4` (16px) | Header: title + action row |
| Inter-column gap | `gap-4` (16px) | Tight kanban density; allows 5-6 columns visible on 1440px width |
| Column min-width | `min-w-[280px]` | Readable card titles; prevents horizontal compression |
| Column max-width | `max-w-[320px]` | Prevents sparse columns from stretching |
| Column internal padding | `p-3` (12px) | Column body — compact |
| Column header padding | `px-3 py-2` (12/8px) | Status title + count pill |
| Card padding | `p-3` (12px) | Linear-style compact card |
| Card min-height | `min-h-[72px]` | Virtualizer `estimateSize: () => 72` exactly matches |
| Inter-card gap (within column) | `gap-2` (8px) | Visible separation without wasted space |
| Card internal row gap | `gap-2` (8px) | title → meta row |
| Horizontal scroll padding | `pr-6` on board container | Prevents last column being flush against viewport |

Exceptions: None. All values on the 4-point grid.

## Interaction Contract

### Drag & Drop

| Event | Library Handler | Action |
|-------|----------------|--------|
| Drag start | `onDragStart={({ active }) => setActiveId(active.id)}` | Set `activeIdRef.current`; freeze WS event queue; during active drag, virtualizer overscan bumps to `items.length` |
| Drag over (cross-column) | `onDragOver` → update `overColumn` local state | Highlight target column header + count pill in accent |
| Drag end | `onDragEnd` → `handleDragEnd(event)` | 1. Compute `{ beforeId, afterId }` via `computeNeighbours()`. 2. Optimistic local reorder. 3. If cross-column: `PATCH /issues/:id { status }`. 4. `POST /issues/:id/reorder { beforeId, afterId }`. 5. Apply server-authoritative position. 6. Flush pending WS events. 7. On error: rollback + `toast.error(t('issues.board.reorderFailed'))`. |
| Drag cancel (Escape) | `onDragCancel` → `setActiveId(null)` + flush queue | Restore original position, announce via a11y |

## WebSocket Reconciliation Contract (UX1 HARD CONSTRAINT)

**State machine (strict):**

```
 IDLE ─── dragStart ──► DRAGGING ─── dragEnd (ok) ──► IDLE (flush queue)
 │                      │                              │
 │                      └── dragCancel ──► IDLE (flush queue)
 │                      │
 │                      └── dragEnd (fail) ──► IDLE (rollback + flush queue)
 │
 └── WS issue:* event ──► apply immediately
```

**Required implementation invariants:**

1. `activeIdRef: RefObject<string | null>` — set on `onDragStart`, cleared AFTER optimistic state commit and server response in `onDragEnd`.
2. `pendingEventsRef: RefObject<WsMessage[]>` — push on WS event when `activeIdRef.current !== null`; otherwise apply immediately.
3. `flushPendingRemoteEvents()` — drain + apply IN ORDER after drag resolution.
4. **Own-echo skip (optimization, optional):** Track `lastLocalMutation: { issueId, position, mutationId }`; when WS `issue:reordered` arrives with matching `issueId + position`, skip the state write (idempotent, saves one render).
5. **IssueCard memo:** `React.memo(IssueCard, (a, b) => a.issue.id === b.issue.id && a.issue.updatedAt === b.issue.updatedAt && a.issue.position === b.issue.position && a.issue.status === b.issue.status && a.isDraggingOverlay === b.isDraggingOverlay)` — prevents rerenders when only sibling cards change.

**WS events consumed (shipped by Phase 17-03):**

| Event type | Handler action |
|------------|---------------|
| `issue:created` | Insert into target status column at position-sorted slot |
| `issue:updated` | Replace matching `issue.id` in-place |
| `issue:deleted` | Remove matching `issue.id` |
| `issue:reordered` | Replace matching `issue.id` with new `position` + `status` |

**Subscription method:** Per research §WebSocket Subscription Semantics Open Question #1 — planner Wave 0 verifies `apps/server/src/ws/index.ts` broadcast semantics. Default assumption: **all authenticated clients receive workspace broadcasts without explicit subscribe**. If wrong, add `subscribeWorkspace(workspaceId)` to `WebSocketContext` (minor change; isolated to one file). Board component should NOT hardcode `DEFAULT_WORKSPACE_ID = 'AQ'` — consume from a central constant if one is exposed, else tolerate the string literal with an inline comment.

## Virtualization Contract (UX4 / UI-03)

| Threshold | Strategy |
|-----------|----------|
| `items.length ≤ 100` per column | Plain `.map()` render — skip virtualizer entirely |
| `items.length > 100` per column | `useVirtualizer({ count, estimateSize: () => 72, overscan: activeId ? items.length : 10 })` |

**Critical rule:** `SortableContext items={items.map(i => i.id)}` receives the **full array of IDs** regardless of virtualization. Only the DOM is windowed.

**During active drag:** Bump `overscan` to `items.length` (effectively disables virtualization for the dragged column) to prevent the dragged card from unmounting. On drag-end, restore `overscan: 10`.

**Verification (test in Wave 0):** Seed 200 issues → open board → `document.querySelectorAll('[data-issue-card]').length ≤ 25` (matches VALIDATION.md row 23-03-01). During drag (assert via Playwright drag action), dragged card persists in DOM.
