Ship `apps/server/src/task-dispatch/hosted-task-worker.ts` — an in-process pump that dispatches queued hosted-instance tasks to the existing gateway via `chat.send`, translates streaming `chat` events into `task_message` rows, and handles cancel / disconnect / ignored-fields per Phase 20 requirements (HOSTED-01, 02, 03, 05, 06).

Purpose: tasks assigned to agents whose runtime is a hosted Aquarium instance must be automatically dispatched through the existing persistent-WS gateway, with live `chat` events translated into `task_message` rows so the UI sees the same streaming shape as the daemon path. Without this worker, hosted tasks stay queued forever.

Output:
- `startHostedTaskWorker(dbOverride?)` + `stopHostedTaskWorker()` — mirrors the Phase 18 task-reaper / Phase 16 offline-sweeper public API.
- A 2 s `setInterval` tick that:
  1. Queries online `hosted_instance` runtimes via JOIN on `runtimes` + `instances` (workspace `'AQ'`).
  2. For each online hosted runtime, calls `isGatewayConnected(instance_id)` → skip silently if false (HOSTED-06).
  3. Otherwise calls `claimTask(runtime_id, kx)` → if a task returns, fork a fire-and-forget `dispatchHostedTask(...)`.
- `dispatchHostedTask` per-task flow:
  1. Resolve `instance_id` via `SELECT instance_id FROM runtimes WHERE id = task.runtimeId`.
  2. Log ignored-fields WARN (HOSTED-05).
  3. Pre-flight `isTaskCancelled(task.id)` — if true, cleanup + return (X5).
  4. `startTask(task.id, kx)` — bail on `{started: false}`.
  5. `registerChatStreamListener` BEFORE `chat.send` so no frames are missed.
  6. Register a REACTIVE cancel watcher: a 2s setInterval that polls isTaskCancelled and routes to handleCancel(task.id) on flip to true (PM6 primary path — independent of stream events).
  7. `waitForChatCompletion(instanceId, sessionKey, 120_000)` in parallel.
  8. `gatewayCall(instanceId, 'chat.send', {sessionKey, message, idempotencyKey: task.id}, 30_000)`.
  9. Listener fires `translatePartsToMessages(payload)` → batched via `appendTaskMessage(taskId, msg)`; ALSO polls `isTaskCancelled` opportunistically (PM6 secondary path — fires faster when frames are arriving).
 10. On `waitForChatCompletion` resolve → `completeTask(task.id, {sessionKey, messageId})`.
 11. On error/throw → `failTask(task.id, 'hosted-dispatch-error: <msg>')` UNLESS aborted by the cancel watcher (then the row is already 'cancelled').
 12. `finally`: unsubscribe, `cancelChatCompletion`, clear cancel watcher interval, clear `inFlight`.
- `translatePartsToMessages(payload, ctx)` — exported for test isolation. Maps ContentPart[] union to PendingTaskMessage[] per 20-RESEARCH §Event-to-Row Translation.
- Test injection hooks `__setHostedWorkerDepsForTests__` + `__resetHostedWorkerState__` (mirror `__setBatcherDbForTests__`).

<cancel_subscription_note>
<!-- Blocker 2 resolution — REACTIVE cancel path design -->

Why polling (not broadcast subscription):
  - `apps/server/src/ws/index.ts` exposes `broadcast()` but NO `onMessage` / `subscribe` /
    `onTaskCancelled` hook. Adding one would cross a subsystem boundary (ws/ internals)
    outside Phase 20's scope.
  - `apps/server/src/services/task-queue-store.ts` emits `task:cancelled` WS frames via
    `broadcastCancelledRows()` but that helper is internal — no subscribable event bus.
  - A 2s setInterval per in-flight task is lightweight: concurrency is bounded by
    `max_concurrent_tasks × |online hosted runtimes|` (practically O(10) in CE). The
    added DB load is trivial (a single SELECT id FROM agent_task_queue WHERE id=?).

Why this path is REACTIVE (not passive polling):
  - It runs independently of the event loop consuming gateway frames. If the gateway
    stops emitting after cancel (the normal behavior), this interval still fires within
    2s and triggers chat.abort — guaranteeing the task row transitions to 'cancelled'
    within a bounded latency window instead of waiting the full 120s for
    waitForChatCompletion to time out.

Future enhancement: if an onTaskCancelled event-bus API is added to ws/index.ts OR
task-queue-store.ts in a later phase, the polling interval can be replaced with a
direct subscription. The handleCancel(taskId) surface stays unchanged.
</cancel_subscription_note>

    STEP 2 — Implement the mapper (HOSTED-03). Place BELOW the module state block:

    ```typescript
    export interface HostedDispatchContext {
      taskId: string;
      workspaceId: string;
      issueId: string;
    }

    /**
     * Translate a gateway chat ChatStreamPayload into zero or more PendingTaskMessage
     * rows. The mapper accepts both gateway-wire formats (camelCase: toolCall/toolResult)
     * AND Claude-style (snake_case: tool_use/tool_result). Unknown part.type values
     * are dropped with a console.warn (HOSTED-03 §Event-to-Row Translation in 20-RESEARCH).
     *
     * Exported so tests can exercise the mapper in isolation.
     */
    export function translatePartsToMessages(
      payload: ChatStreamPayload,
      ctx: HostedDispatchContext,
    ): PendingTaskMessage[] {
      const raw = payload.message?.content ?? payload.content;
      if (raw == null) return [];
      if (typeof raw === 'string') {
        // String fallback — gateway occasionally sends terminal content as a bare string.
        return [{
          type: 'text',
          content: raw,
          workspaceId: ctx.workspaceId,
          issueId: ctx.issueId,
        }];
      }
      if (!Array.isArray(raw)) return [];
      const out: PendingTaskMessage[] = [];
      for (const partRaw of raw) {
        if (typeof partRaw !== 'object' || partRaw === null) continue;
        const part = partRaw as Record<string, unknown>;
        const t = typeof part.type === 'string' ? part.type : '';
        switch (t) {
          case 'text':
          case 'output_text':
          case 'input_text':
            out.push({
              type: 'text',
              content: typeof part.text === 'string' ? part.text : '',
              workspaceId: ctx.workspaceId,
              issueId: ctx.issueId,
            });
            break;
          case 'thinking':
            out.push({
              type: 'thinking',
              content: typeof part.thinking === 'string'
                ? part.thinking
                : (typeof part.text === 'string' ? part.text : ''),
              workspaceId: ctx.workspaceId,
              issueId: ctx.issueId,
            });
            break;
          case 'toolCall':
          case 'tool_use': {
            const name = typeof part.name === 'string' ? part.name : '';
            const input = part.arguments ?? part.input ?? null;
            const id = typeof part.id === 'string' ? part.id : undefined;
            out.push({
              type: 'tool_use',
              tool: name,
              input,
              metadata: id ? { toolUseId: id } : {},
              workspaceId: ctx.workspaceId,
              issueId: ctx.issueId,
            });
            break;
          }
          case 'toolResult':
          case 'tool_result': {
            const toolUseId = part.tool_use_id ?? part.toolUseId ?? null;
            const isError = Boolean(part.is_error ?? part.isError ?? false);
            out.push({
              type: 'tool_result',
              tool: null,
              output: part.content ?? part.result ?? null,
              metadata: { toolUseId, isError },
              workspaceId: ctx.workspaceId,
              issueId: ctx.issueId,
            });
            break;
          }
          case 'image':
          case 'image_url':
          case 'file':
            console.warn(
              `[hosted-task-worker] dropping unsupported content part type '${t}' for task ${ctx.taskId}`,
            );
            break;
          default:
            console.warn(
              `[hosted-task-worker] unknown content part type '${t}' for task ${ctx.taskId}`,
            );
        }
      }
      return out;
    }
    ```

    ```typescript
     * Start the hosted-task worker tick. Idempotent (subsequent calls are no-ops).
     * Fires an initial tick synchronously on start so a cold server does not wait
     * 2s before dispatching the first hosted task.
     *
     * The optional `dbOverride` parameter is the Knex instance tests inject via
     * setupTestDb() — production callers (server-core Step 9d) pass no argument
     * and the worker uses the app singleton.
     */
    export function startHostedTaskWorker(dbOverride?: Knex): void {
      if (tickHandle) return;
      if (dbOverride) activeDb = dbOverride;

      // Initial tick — do not wait TICK_MS before first dispatch.
      tick(activeDb).catch((err) => {
        console.warn(
          '[hosted-task-worker] initial tick failed:',
          err instanceof Error ? err.message : String(err),
        );
      });

      tickHandle = setInterval(() => {
        tick(activeDb).catch((err) => {
          console.warn(
            '[hosted-task-worker] tick failed:',
            err instanceof Error ? err.message : String(err),
          );
        });
      }, TICK_MS);

      console.log('[hosted-task-worker] started (2s tick interval)');
    }

    /**
     * Stop the hosted-task worker tick AND await any in-flight dispatches so
     * graceful shutdown does not leave orphan chat.send RPCs on the wire.
     */
    export async function stopHostedTaskWorker(): Promise<void> {
      if (tickHandle) {
        clearInterval(tickHandle);
        tickHandle = null;
      }
      // Snapshot + wait. Dispatch catch-all ensures none of these ever reject.
      const pending = [...dispatchPromises];
      await Promise.allSettled(pending);
      console.log('[hosted-task-worker] stopped');
    }
    ```

    ```typescript
    async function dispatchHostedTask(
      task: ClaimedTask,
      instanceId: string,
      kx: Knex,
    ): Promise<void> {
      warnIgnoredFields(task);

      // X5 pre-flight: user may have cancelled between enqueue and our claim.
      if (await isTaskCancelled(task.id, kx)) {
        console.log(`[hosted-task-worker] task ${task.id} cancelled before dispatch — skipping chat.send`);
        return;  // task row already 'cancelled'; completeTask/failTask would return {discarded:true}
      }

      // Transition dispatched → running. startTask guards against race (returns {started:false}
      // if the row moved to cancelled/failed between claim and here).
      const startedResult = await startTask(task.id, kx);
      if (!startedResult.started) {
        console.log(
          `[hosted-task-worker] task ${task.id} not in dispatched state (now ${startedResult.status}) — skipping`,
        );
        return;
      }

      const sessionKey = `task:${task.id}`;
      const abort = new AbortController();
      // Dedupe set for final-frame duplicates (gateway may repeat streaming content in final).
      // Signature keys on `${type}:${JSON.stringify(input|output|content)}`.
      const seenSignatures = new Set<string>();

      const ctx: HostedDispatchContext = {
        taskId: task.id,
        workspaceId: task.workspaceId,
        issueId: task.issueId,
      };

      // --- PM6 REACTIVE cancel watcher -----------------------------------
      // Why a setInterval (not broadcast subscription): ws/index.ts exposes
      // `broadcast()` but NO subscribe/onTaskCancelled hook. A 2s poll per
      // in-flight task is lightweight (bounded by max_concurrent_tasks ×
      // |online hosted runtimes|) and guarantees cancel is detected within
      // CANCEL_POLL_MS REGARDLESS of whether the gateway stops emitting
      // frames after the cancel. If the gateway stops (the typical behavior),
      // the opportunistic poll in the stream listener never fires — only
      // this reactive watcher breaks the 120s waitForChatCompletion block.
      const cancelWatcher: ReturnType<typeof setInterval> = setInterval(() => {
        void isTaskCancelled(task.id, kx).then((cancelled) => {
          if (cancelled) handleCancel(task.id);
        }).catch((err) => {
          // Swallow errors — reactive poll MUST NOT crash the dispatch.
          console.warn(
            `[hosted-task-worker] cancel poll failed for task ${task.id}:`,
            err instanceof Error ? err.message : String(err),
          );
        });
      }, CANCEL_POLL_MS);

      // Subscribe BEFORE chat.send so no frames are missed (§Pitfall 2 in 20-RESEARCH).
      const unsubscribe = deps.registerChatStreamListener(instanceId, sessionKey, (payload) => {
        // Capture runId for chat.abort (§Cancel/Abort Semantics — gateway uses runId
        // to correlate abort with the in-flight run when present).
        if (payload.runId) {
          const entry = inFlight.get(task.id);
          if (entry && !entry.runId) entry.runId = payload.runId;
        }
        // OPPORTUNISTIC cancel poll — piggy-backs on event arrival per §Cancel/Abort.
        // Complements the REACTIVE watcher above by firing faster when frames arrive.
        void isTaskCancelled(task.id, kx).then((cancelled) => {
          if (cancelled) handleCancel(task.id);
        });
        const msgs = translatePartsToMessages(payload, ctx);
        for (const m of msgs) {
          const sig = `${m.type}:${m.content ?? ''}:${JSON.stringify(m.input ?? null)}:${JSON.stringify(m.output ?? null)}`;
          if (seenSignatures.has(sig)) continue;  // Final-frame dedupe.
          seenSignatures.add(sig);
          deps.appendTaskMessage(task.id, m);
        }
      });

      inFlight.set(task.id, {
        abort, sessionKey, instanceId, unsubscribe,
        cancelWatcher, startedAt: Date.now(),
      });

      // Waiter BEFORE chat.send — avoid missing the final frame on a fast gateway.
      const completion = deps.waitForChatCompletion(instanceId, sessionKey, CHAT_WAIT_TIMEOUT_MS);
      completion.catch(() => { /* swallow — handled below */ });

      try {
        const prompt = buildPromptFromTask(task);

        await deps.gatewayCall(instanceId, 'chat.send', {
          sessionKey,
          message: prompt,
          idempotencyKey: task.id,        // §Pitfall 3 — use task.id (stable UUID), NOT randomUUID()
        }, CHAT_SEND_TIMEOUT_MS);

        const result = await completion;

        // One last pass for the final frame (catches any parts only present in the 'final' payload).
        const finalParts = translatePartsToMessages(
          { sessionKey, state: 'final', content: result.content, messageId: result.messageId, role: result.role },
          ctx,
        );
        for (const m of finalParts) {
          const sig = `${m.type}:${m.content ?? ''}:${JSON.stringify(m.input ?? null)}:${JSON.stringify(m.output ?? null)}`;
          if (seenSignatures.has(sig)) continue;
          seenSignatures.add(sig);
          deps.appendTaskMessage(task.id, m);
        }

        // completeTask handles PM6 — returns {discarded:true, status:'cancelled'} if user
        // cancelled between final and here; we don't need to branch on that — it is idempotent.
        await completeTask(task.id, { sessionKey, messageId: result.messageId }, kx);
      } catch (err) {
        const msg = err instanceof Error ? err.message : String(err);
        if (abort.signal.aborted) {
          // Cancel path already flipped the task row to 'cancelled' via cancelTask;
          // do NOT call failTask (it would return {discarded:true}, but emitting
          // 'hosted-dispatch-error' would overwrite the cancel cleanliness).
          return;
        }
        // Fire-and-forget — failTask is idempotent for already-terminal states.
        await failTask(task.id, `hosted-dispatch-error: ${msg}`, kx);
      } finally {
        unsubscribe();
        deps.cancelChatCompletion(instanceId, sessionKey);
        clearInterval(cancelWatcher);  // REACTIVE watcher cleanup
        inFlight.delete(task.id);
      }
    }

    function buildPromptFromTask(task: ClaimedTask): string {
      // Canonical prompt assembly: instructions + issue title + description + trigger comment.
      // Kept simple per 20-RESEARCH (no new research required — text concatenation is deterministic).
      const parts: string[] = [];
      if (task.agent.instructions) parts.push(task.agent.instructions);
      parts.push(`Issue #${task.issue.issueNumber}: ${task.issue.title}`);
      if (task.issue.description) parts.push(task.issue.description);
      if (task.triggerCommentContent) parts.push(`User: ${task.triggerCommentContent}`);
      return parts.join('\n\n');
    }

    /**
     * React to a cancel signal. Idempotent — if the task is not in-flight (already
     * completed, or cancel arrived after finally{} cleanup), this is a no-op (X6).
     *
     * Invoked from TWO paths:
     *   1. REACTIVE: the per-task CANCEL_POLL_MS setInterval (primary — stream-independent).
     *   2. OPPORTUNISTIC: the stream-listener's isTaskCancelled poll (fallback — faster
     *      when frames are arriving, but silent when the gateway stops emitting).
     */
    function handleCancel(taskId: string): void {
      const entry = inFlight.get(taskId);
      if (!entry) return;
      if (entry.abort.signal.aborted) return;  // Already processed — idempotent no-op.
      entry.abort.abort();
      // Best-effort gateway abort — web UI ignores failures (§Cancel/Abort Semantics).
      // Include runId when captured from a prior stream frame.
      const abortParams: Record<string, unknown> = { sessionKey: entry.sessionKey };
      if (entry.runId) abortParams.runId = entry.runId;
      deps.gatewayCall(entry.instanceId, 'chat.abort', abortParams, CHAT_ABORT_TIMEOUT_MS)
        .catch((err) => console.warn(
          '[hosted-task-worker] chat.abort failed:',
          err instanceof Error ? err.message : String(err),
        ));
      deps.cancelChatCompletion(entry.instanceId, entry.sessionKey);
      // Do NOT inFlight.delete or clearInterval here — the finally{} in
      // dispatchHostedTask handles both. This keeps cleanup single-path.
    }
    ```
