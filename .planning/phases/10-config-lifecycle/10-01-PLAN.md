Rewrite `patchGatewayConfig()` (lines ~736-845) to implement the gateway-first pattern. Keep the same function signature (`instanceId, userId, configPatch, note?`).

**Key changes:**

1. **Add import** for `gatewayCall` from `../agent-types/openclaw/gateway-rpc.js` at the top of the file.

2. **Stopped instances (CFG-02):** Move the stopped-instance check BEFORE any DB write. If `instance.status !== 'running'` or no `controlEndpoint`, deep-merge `configPatch` into existing config and persist to DB via `updateInstanceConfig`. Return early. This is the ONLY path that writes DB without gateway confirmation.

3. **Running instances -- gateway-first (CFG-01):** For running instances, do NOT write to DB before the gateway confirms. The flow is:
   - Keep `safeAutoSnapshot` and `validateConfigPatch` calls at the top (before gateway interaction)
   - Enter a retry loop (MAX_RETRIES = 3):
     a. Call `gatewayCall(instanceId, 'config.get', {})` to get current `baseHash` (CFG-03)
     b. Call `gatewayCall(instanceId, 'config.patch', { raw: JSON.stringify(configPatch), baseHash, note: note || 'Platform config update', restartDelayMs: 2000 })` (CFG-04)
     c. On stale hash error (message includes `'config changed since last load'` or `'CONFLICT'`): if attempts remain, `continue` to retry from step (a)
     d. On rate limit error (message includes `'rate limit'`): parse `retryAfterMs` from message using `/retry after (\d+)s/` regex, `await` a delay of `parsedMs + 1000` (buffer), then `continue` to retry from step (a) (CFG-05)
     e. On any other error: throw immediately (locked decision: fail visibly)
   - After successful `config.patch`:
     f. Call `gatewayCall(instanceId, 'config.get', {})` again for authoritative hash (CFG-07). Wrap in try/catch -- if read-back fails (gateway may be restarting due to SIGUSR1), fall back to `baseHash` from step (a). Log a warning on fallback.
     g. Deep-merge `configPatch` into existing `instance.config` using `deepMerge()` (platform's version with credential placeholders intact -- NEVER store redacted config from gateway)
     h. Persist to DB synchronously: update `config` (JSON.stringify of merged config), `config_hash` (authoritative hash from step f), `updated_at`, and sync `billing_mode`/`name` columns if applicable (copy the column-sync logic from `updateInstanceConfig`)
   - If all retries exhausted: throw `'Config patch failed after max retries'`

4. **Remove all old code:** Delete the `reseedConfigFiles(instanceId)` call, the `engine.readFile` block that reads openclaw.json from disk, the `adapter.translateRPC` calls (replaced by `gatewayCall`), and the `getAgentType`/`getRuntimeEngine` usage within this function. The old "Gateway push failure is non-critical" comment and swallowed error at the end (lines 840-844) are WRONG under gateway-first semantics -- gateway failure MUST propagate.

5. **Keep `deepMerge` function** (lines 721-734) unchanged -- it is still needed.

6. **Remove the `reseedConfigFiles` import dependency for patchGatewayConfig** -- the function no longer calls it. However, `reseedConfigFiles` is still exported and used by health-monitor and startInstanceAsync, so do NOT delete the function itself.

**Important: Do NOT change `updateInstanceConfig`** -- it is still used by the stopped-instance path and other callers.

**Validation note:** The `validateConfigPatch` call with `skipFullSchemaValidation: true` should remain in place for basic sanity checking. However, move it before the running/stopped check so both paths get validation. The `fetchSchema` closure can be simplified: for running instances it attempts `gatewayCall(instanceId, 'config.schema', {})` instead of the old `adapter.translateRPC` pattern; for stopped instances it returns null (no schema available).
