<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser→server | Cookie-JWT `api.get` to `/api/runtimes` via existing `requireAuth` middleware |
| server→browser | Runtime objects rendered as React text children; `deviceInfo` + `metadata` rendered via `JSON.stringify(... , null, 2)` inside `<pre>` — auto-escaped |
| client-side state | Kind filter + search query + detail runtime — all ephemeral React state, no browser storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-02-01 | Tampering (XSS via `deviceInfo` / `metadata` JSON) | RuntimeDetailSheet + RuntimeList tooltip | mitigate | All JSON rendered via `{JSON.stringify(obj, null, 2)}` as React text children inside `<pre>` — auto-escaped. No `dangerouslySetInnerHTML` (CI guard from 25-00 blocks PRs that introduce it). |
| T-25-02-02 | Tampering (URL query injection via `?kind=`) | RuntimesPage + useSearchParams | mitigate | Initial filter read ONLY accepts enum values from `RuntimeKind` union; any other value silently coerced to `'all'`. No string interpolation into DOM/URL elsewhere. |
| T-25-02-03 | Information Disclosure (runtime hostname leak in tooltip) | RuntimeList device tooltip | accept | `deviceInfo.hostname` is user-controlled telemetry from own daemon; CE is single-tenant — no cross-tenant leak. |
| T-25-02-04 | Clickjacking (Sheet overlay) | RuntimeDetailSheet | mitigate | shadcn Sheet (`@radix-ui/react-dialog`) ships focus trap + overlay + aria-modal + Escape close by default. |
| T-25-02-05 | DoS (30-second polling under rapidly-refreshing runtime list) | useRuntimes | accept | Single server call per 30s; server endpoint is a simple SELECT with JOIN; CE is single-user — no amplification risk. |
| T-25-02-06 | Spoofing (kind filter reflects untrusted URL) | RuntimesPage | mitigate | Mitigated by strict enum check on `?kind=` param (T-25-02-02); attacker-controlled value cannot trigger anything beyond filter state. |
</threat_model>
