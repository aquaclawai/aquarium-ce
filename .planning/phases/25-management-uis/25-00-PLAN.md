**Why `! grep` and not `if grep; then exit 1; fi`:** grep exits 2 on certain errors (e.g., broken permission, unreadable file). `if grep; then exit 1; fi` treats exit 2 as "not found" (falsy) and silently passes â€” a security-guard failure mode. The `! grep` form uses Bash's logical-not: it passes ONLY when grep exits 1 (no matches). Exit codes 0 or 2 both make the step fail, as intended for a bulletproof MGMT-03 guard.
