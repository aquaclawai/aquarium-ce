  <action>
Create apps/server/src/services/trust-store.ts following the service pattern from skill-store.ts (import getAdapter, db from adapter.js).

Implement the following exported functions:

1. `computeTrustTier(source: PluginSource | ExtensionSkillSource, signals: TrustSignals | null): TrustTier`
   - If source.type === 'bundled' -> return 'bundled'
   - If signals is null -> return 'unscanned' (no metadata available)
   - If signals.virusTotalPassed === null -> return 'unscanned'
   - If signals.virusTotalPassed === false -> return 'unscanned' (scan failed = block always)
   - If signals.verifiedPublisher && signals.downloadCount > 100 && signals.ageInDays > 90 && signals.virusTotalPassed === true -> return 'verified'
   - Otherwise -> return 'community'

2. `evaluateTrustPolicy(instanceId: string, extensionId: string, extensionKind: ExtensionKind, source: PluginSource | ExtensionSkillSource, signals: TrustSignals | null): Promise<TrustEvaluation>`
   - Compute tier via computeTrustTier
   - If tier is 'bundled' or 'verified' -> decision = 'allow', blockReason = null
   - If tier is 'unscanned' -> decision = 'block', blockReason = 'Security scan not available or failed. This extension cannot be installed.' (no override possible for unscanned)
   - If tier is 'community':
     - Check for existing trust override via getTrustOverride(instanceId, extensionId, extensionKind)
     - If override exists -> decision = 'allow', include override in result
     - If no override -> decision = 'block', blockReason = 'Community extension — not verified. An admin must approve before installation.'
   - Return full TrustEvaluation object

3. `createTrustOverride(instanceId: string, extensionId: string, extensionKind: ExtensionKind, reason: string, userId: string, credentialAccessAcknowledged: boolean): Promise<TrustOverride>`
   - Validate credentialAccessAcknowledged === true, throw Error('Credential access acknowledgment required') if false
   - INSERT into trust_overrides with all fields, using adapter.generateId() for id
   - Use ON CONFLICT (instance_id, extension_id, extension_kind) DO UPDATE to allow re-override with new reason
   - Return the TrustOverride row (query back after upsert)

4. `getTrustOverride(instanceId: string, extensionId: string, extensionKind: ExtensionKind): Promise<TrustOverride | null>`
   - SELECT from trust_overrides WHERE instance_id, extension_id, extension_kind
   - Map DB row to TrustOverride type (snake_case -> camelCase)
   - Return null if not found

5. `getTrustOverridesForInstance(instanceId: string): Promise<TrustOverride[]>`
   - SELECT all from trust_overrides WHERE instance_id
   - Map rows to TrustOverride[]

Helper: `mapTrustOverrideRow(row: Record<string, unknown>): TrustOverride` for consistent mapping.

IMPORTANT: Server imports must use .js extension (e.g., `import { getAdapter, db } from '../db/adapter.js';`).
