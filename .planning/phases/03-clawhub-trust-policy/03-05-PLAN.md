3. Add version info section ABOVE the credential form (after the header, before the hint):
   - Only render if lockedVersion is non-null
   - Display: "Version: {lockedVersion} (pinned)" using i18n key extensions.version.pinned
   - Display truncated integrity hash: "Integrity: sha512-{first 16 chars}...{last 8 chars}" using i18n key extensions.version.integrity
     - If integrityHash is null, show "Integrity: not available"
   - Two-step upgrade flow (per CONTEXT.md decision):
     - "Check for Updates" button sends PUT /instances/{instanceId}/{extensionKind}s/{extensionId}/upgrade with body `{ dryRun: true }`
     - If upToDate: show "Up to date" green text
     - If not upToDate: show "Current: v{current} -> Latest: v{newVersion}" with "Upgrade" button
     - "Upgrade" button sends PUT with `{ dryRun: false }` to perform the actual upgrade
     - The executor must add `dryRun` support to the upgrade endpoint (plan 03-03): if `req.body.dryRun === true`, query ClawHub for latest version and return `{ upToDate, currentVersion, newVersion }` without performing the upgrade; if `dryRun` is false or omitted, perform the upgrade as specified in plan 03-03
   - "Check for Updates" flow:
     1. Click button -> PUT /upgrade with { dryRun: true }
     2. If upToDate: show "Up to date" green text
     3. If not upToDate: show "Current: v{current} -> Latest: v{newVersion}" with "Upgrade" button
   - "Upgrade" button:
     1. Click -> PUT /upgrade with { dryRun: false }
     2. Show spinner during upgrade
     3. On success: update version display, show success message, call onSaved() to refresh parent
     4. On error: show error message
     5. For plugins: show note that restart is required (i18n key extensions.version.restartRequired)

4. Add trust override audit trail section BELOW the credential form:
   - Only render if trustOverride is non-null
   - Display: "Admin-approved by [userId] on [createdAt formatted]: [reason]"
   - Use i18n key extensions.trust.auditTrail
   - Style as a subtle info box (.credential-panel__audit)
