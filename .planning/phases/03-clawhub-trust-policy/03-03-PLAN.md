<task type="auto">
  <name>Task 2: Extend catalog routes with ClawHub merging, trust signals, and upgrade endpoints</name>
  <files>apps/server/src/routes/skills.ts, apps/server/src/routes/plugins.ts</files>
  <action>
Extend both route files to merge ClawHub results into catalog endpoints and add upgrade endpoints.

**plugins.ts — extend GET /:id/plugins/catalog:**

1. Read query params: `search`, `category`, `page` (default '0'), `limit` (default '20')
2. Existing code fetches bundled catalog from `plugins.list` RPC. Keep that as-is.
3. After bundled results, if instance is running and controlEndpoint exists, also call `searchClawHub(controlEndpoint, authToken, { query: search, category, kind: 'plugin', offset: page * limit, limit })` from marketplace-client.ts
4. For each bundled catalog entry, add `trustTier: 'bundled' as TrustTier` and `trustSignals: undefined`
5. For each ClawHub entry, compute trust tier via `computeTrustTier(source, entry.trustSignals)` and map to PluginCatalogEntry shape with added trustTier and trustSignals fields
6. Merge: bundled entries first, then ClawHub entries. Deduplicate by id (bundled wins if same id exists in both)
7. Return merged list in existing ApiResponse format
8. If searchClawHub fails, soft-log warning and return only bundled results (same resilience pattern as plugins.list RPC failure)

**plugins.ts — add PUT /:id/plugins/:pluginId/upgrade:**

IMPORTANT: Define this route BEFORE the existing PUT /:id/plugins/:pluginId route to prevent Express route capture. The path is `/:id/plugins/:pluginId/upgrade`.

1. Verify instance exists, belongs to user, is running
2. Read existing plugin row via getPluginById -- must exist and be in 'active' or 'installed' or 'disabled' status
3. Call getClawHubExtensionInfo(controlEndpoint, authToken, pluginId, 'plugin') to get latest version
4. If no ClawHub info or same version as lockedVersion: return 200 with `{ ok: true, data: { upToDate: true, currentVersion: plugin.lockedVersion } }`
5. If newer version available: acquire lock, call plugins.install RPC with the new version, update locked_version and integrity_hash in DB, release lock
6. If plugin was 'active': trigger activation (restart) for the new version
7. Return 200 with `{ ok: true, data: { upToDate: false, previousVersion, newVersion, plugin } }`

**skills.ts — same pattern:**

1. Extend GET /:id/skills/catalog with ClawHub merging (same approach as plugins)
2. Add PUT /:id/skills/:skillId/upgrade endpoint:
   - Similar to plugin upgrade but NO restart needed (skills don't require restart)
   - Acquire lock, call skills.install RPC with new version spec, update locked_version and integrity_hash, release lock
   - Define this route BEFORE the existing PUT /:id/skills/:skillId route

Import searchClawHub, getClawHubExtensionInfo from marketplace-client.ts and computeTrustTier from trust-store.ts.
