**OAuth Connect section** (render above the manual credential form when `supportsOAuth` is true):
- Show a "Connect with {provider}" button styled with `.btn--primary`
- When `requiresReAuth` is true, show an alert: "This extension requires re-authentication after import. Click Connect to authenticate."
- On click: call `api.post(\`/instances/${instanceId}/oauth-proxy/initiate\`, { extensionId, extensionKind, provider: oauthProvider })`
- On success: open `data.authUrl` in a popup window: `window.open(authUrl, 'oauth-popup', 'width=600,height=700')`
- Listen for `message` events on the window for the callback postMessage from the popup:
  ```typescript
  useEffect(() => {
    const handler = (event: MessageEvent) => {
      if (event.data?.type === 'oauth-callback' && event.data?.extensionId === extensionId) {
        if (event.data.status === 'success') {
          setOauthConnected(true);
          setOauthConnecting(false);
          onSaved(); // Refresh extension list
        } else {
          setOauthError(event.data.error || t('extensions.oauth.failed'));
          setOauthConnecting(false);
        }
      }
    };
    window.addEventListener('message', handler);
    return () => window.removeEventListener('message', handler);
  }, [extensionId, onSaved, t]);
  ```
- When `oauthConnected` is true, show a success message: "{provider} authenticated" with a checkmark
- When `oauthError` is set, show error message with a "Try again" button
