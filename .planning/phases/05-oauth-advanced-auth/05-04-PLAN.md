<objective>
Fix the OAuth callback userId bug that prevents the entire OAuth token exchange flow from completing.

Purpose: The callback handler at line 179 passes `session.instanceId` (an instance UUID) as the `userId` parameter to `getInstance(id, userId)`. Since an instance UUID never matches a user UUID, `getInstance` always returns null, making the token exchange RPC call, sentinel credential write, and extension status update all unreachable. This single bug blocks requirements OAUTH-01 and OAUTH-02.

Output: A working OAuth callback that correctly identifies the user, allowing token exchange, credential sentinel row creation, and extension activation.
</objective>
