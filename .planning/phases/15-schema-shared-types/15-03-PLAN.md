## Critical Design Notes

1. **`runtime_id` is SET NULL on delete** (NOT CASCADE). Agents are audit data â€” a runtime going away doesn't delete the agent's history; the agent simply becomes "unassigned" and tasks can't dispatch until a new runtime is set. This is a deliberate departure from ROADMAP owned-pitfall ST4 (which dictates CASCADE for runtimeâ†’TASK); for runtimeâ†’AGENT we use SET NULL because agents outlive runtimes. This matches research PITFALLS.md Â§ST4 prevention: "ON DELETE SET NULL on agents.runtime_id".
2. **`max_concurrent_tasks` default 6, CHECK 1..16** per SCH-03 requirement â€” enforced via trigger on INSERT + UPDATE.
3. **`custom_env` stored as JSON plain text** â€” encryption at rest deferred to v1.5 per research "Resolved Decision #18". Schema must not assume encryption; a plain `text` (JSON) column is correct.
4. **`archived_at` and `archived_by`** are nullable timestamps/FKs; soft-delete is performed by setting both â€” no CASCADE from here because archival is reversible.
