4. **Add** a private `fetchClawHub` helper (adapted from plugin's version):
```typescript
async function fetchClawHub(endpoint: string, params: Record<string, unknown>): Promise<unknown> {
  if (!config.clawHubApiUrl) return null;
  const url = new URL(endpoint, config.clawHubApiUrl);
  const resp = await fetch(url.toString(), {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(params),
    signal: AbortSignal.timeout(15_000),
  });
  if (!resp.ok) return null;
  return resp.json();
}
```
Note: uses `config.clawHubApiUrl` (not `process.env` directly, per project conventions).

5. **Rewrite `searchClawHub`** -- remove `instanceId` parameter. New signature:
```typescript
export async function searchClawHub(
  params: { query?: string; category?: string; kind?: ExtensionKind; offset?: number; limit?: number },
): Promise<{ entries: ClawHubCatalogEntry[]; total: number; hasMore: boolean }>
```
Implementation:
- Try remote ClawHub API first via `fetchClawHub('/api/search', rpcParams)`.
- If remote returns a valid response, parse entries through the existing `parseClawHubEntry` function.
- If remote is unavailable (`fetchClawHub` returns null) or response shape is unexpected, fall back to filtering `BUILTIN_REGISTRY` in-memory (same filter logic as the plugin's `clawhub.search` handler: filter by kind, category, query match on name/description/id, apply offset+limit).
- Wrap remote call in try/catch -- on any error, soft-fail to BUILTIN_REGISTRY fallback with a `console.warn`.
- Keep existing return shape `{ entries, total, hasMore }`.

6. **Rewrite `getClawHubExtensionInfo`** -- remove `instanceId` parameter. New signature:
```typescript
export async function getClawHubExtensionInfo(
  extensionId: string,
  kind: ExtensionKind,
): Promise<ClawHubCatalogEntry | null>
```
Implementation:
- Try remote first via `fetchClawHub('/api/info', { extensionId, kind })`.
- Parse through `parseClawHubEntry`.
- If unavailable, fall back to `BUILTIN_REGISTRY.find(e => e.id === extensionId && (!kind || e.kind === kind))`.
- Wrap remote call in try/catch with `console.warn` on error.

7. **Keep** the existing `parseClawHubEntry` helper function unchanged -- it validates response shapes and remains useful for parsing remote ClawHub API responses.
