const filteredCatalog = availableCatalog.filter(entry => {

**3. Create RestartBanner.tsx** — gateway restart progress banner

```typescript
interface RestartBannerProps {
  pluginName: string;
  instanceId: string;
  pluginId: string;
  onComplete: (success: boolean, error?: string) => void;
}
```

Per user decision from CONTEXT.md:
- Full-width banner at top of Extensions tab: "Gateway restarting for plugin X..." with CSS spinner
- All plugin actions disabled tab-wide during restart (parent ExtensionsTab controls this via `isRestarting` state)
- Polling: GET `/instances/${instanceId}/plugins/${pluginId}` every 2s until status changes from 'installed'/'pending' to 'active' or 'failed'
- On status change: call onComplete(true) for active, onComplete(false, errorMessage) for failed
- Use `useEffect` with `setInterval` for polling. Clean up interval on unmount.

Wire into ExtensionsTab: when activatePlugin is called, show the RestartBanner and set `isRestarting = true`. Pass `onComplete` that clears the banner and refreshes data.
