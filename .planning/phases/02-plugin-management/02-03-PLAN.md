**Create `ExtensionRow.tsx`** — a shared component that handles both skills and plugins. Based on the existing SkillRow.tsx but with an `extensionKind` prop for conditional rendering.

```typescript
interface ExtensionRowProps {
  extensionKind: 'plugin' | 'skill';
  extensionId: string;       // pluginId or skillId
  extensionName: string;     // Display name (may differ from ID)
  status: ExtensionStatus;
  enabled: boolean;
  errorMessage: string | null;
  onToggle: (id: string, enabled: boolean) => void;
  onUninstall: (id: string) => void;
  onConfigure: (id: string) => void;
  onActivate?: (id: string) => void;  // Plugin-specific: only shown for 'installed' plugins
  disabled: boolean;
  activating?: boolean;  // Plugin-specific: shows spinner on Activate button
}
```

Key differences from SkillRow:
- When `extensionKind === 'plugin'` AND `status === 'installed'`: show an "Activate" button (per user decision) where the toggle normally is. The Activate button calls `onActivate`. Once activated (status='active'), the button becomes the normal toggle.
- The gear icon for credentials always shows (works for both kinds via CredentialConfigPanel).
- Status dot uses the same `getStatusDotClass` helper (shared).
- The uninstall X button works the same for both.

Reuse all CSS classes from ExtensionsTab.css (`.skill-row`, `.skill-row__*`, `.status-dot--*`, `.toggle-switch`, `.icon-button--danger`). No new CSS needed since plugins use the same row layout.

**Create `CatalogExtensionRow.tsx`** — shared catalog row for both plugins and skills.

interface CatalogExtensionRowProps {

  disabled: boolean;

For plugins, show capabilities badges (e.g., "channel", "model-provider"). For skills, show binary requirements. Same Install button pattern.

If `extensionKind === 'plugin'`, add a small note below the Install button: a line that says t('extensions.plugins.requiresRestart') so the user knows installation of a plugin involves a gateway restart (per user decision from CONTEXT.md).

**Update existing SkillRow.tsx and CatalogSkillRow.tsx**: These files should be kept as thin wrappers that delegate to the shared components, OR the imports in ExtensionsTab.tsx should be updated to use the new shared components directly.
