**Create `apps/server/src/routes/plugins.ts`** following the exact pattern of the existing `routes/skills.ts`. Use Express Router, export default.

Seven route handlers:

1. **GET /:id/plugins** — List managed plugins + gateway built-ins
   - Call `getPluginsForInstance(instanceId)` for managed plugins
   - If instance is running (check instance status from DB or req context), call `GatewayRPCClient.call('plugins.list', {}, 30_000)` for gateway built-ins
   - Return `{ ok: true, data: { managed: InstancePlugin[], gatewayBuiltins: GatewayExtensionInfo[] } }`
   - When instance is not running, return gatewayBuiltins as empty array (same pattern as GET /skills)

2. **GET /:id/plugins/catalog** — Browse available plugins
   - Call `GatewayRPCClient.call('plugins.list', {}, 30_000)` to gateway
   - Parse response into `PluginCatalogEntry[]` format
   - Support query params: `search` (filter by name/description), `category` (filter by category)
   - Return `{ ok: true, data: PluginCatalogEntry[] }`

3. **GET /:id/plugins/:pluginId** — Single plugin read (used by RestartBanner polling)
   - Call `getPluginById(instanceId, pluginId)`
   - If not found, return 404: `{ ok: false, error: 'Plugin not found' }`
   - Return `{ ok: true, data: plugin }`
   - No lock needed — this is a simple read.

4. **POST /:id/plugins/install** — Install plugin artifact
   - Body: `{ pluginId: string, source: string }` (source is 'bundled', 'clawhub', or 'npm')
   - Parse source into PluginSource type object
   - Call `installPlugin(instanceId, pluginId, source, userId)` — userId from req (auth middleware provides it)
   - Catch LockConflictError -> return 409 with `{ ok: false, error: message, activeOperation }`
   - Return `{ ok: true, data: { plugin, requiredCredentials } }`

5. **POST /:id/plugins/:pluginId/activate** — Activate installed plugin (triggers restart)
   - Call `activatePlugin(instanceId, pluginId, userId)`
   - Catch LockConflictError -> 409
   - Return `{ ok: true, data: plugin }`

6. **PUT /:id/plugins/:pluginId** — Toggle enable/disable
   - Body: `{ enabled: boolean }`
   - If enabled: call `enablePlugin(instanceId, pluginId, userId)`
   - If !enabled: call `disablePlugin(instanceId, pluginId, userId)`
   - Catch LockConflictError -> 409
   - Return `{ ok: true, data: plugin }`

7. **DELETE /:id/plugins/:pluginId** — Uninstall plugin
   - Call `uninstallPlugin(instanceId, pluginId, userId)`
   - Catch LockConflictError -> 409
   - Return `{ ok: true }`

**IMPORTANT: Route ordering.** Place the `GET /:id/plugins/catalog` route BEFORE `GET /:id/plugins/:pluginId` so that `/catalog` is not captured as a pluginId parameter. Alternatively, use a regex constraint on pluginId. The safest approach: define catalog first, then :pluginId routes.

**Authentication:** All routes assume auth middleware has already set `req.user` with `{ id: string }`. Access the userId as `(req as any).user?.id` or follow the existing pattern in skills.ts for getting the userId.

Actually, check how skills.ts gets the userId — likely from `req.params` or a middleware. Follow that exact pattern.
