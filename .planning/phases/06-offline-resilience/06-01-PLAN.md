<objective>
Implement plugin/skill artifact caching so gateway restarts and air-gapped deployments can rebuild installed extensions without hitting the external registry.

Purpose: Reduce dependency on external registries during restart/rebuild. After first successful install, the artifact tarball is cached locally inside the container's persistent volume. Subsequent rebuilds prefer the cache, falling back to registry only on cache miss.

Output: artifact-cache.ts service, modified install flows with cache-after-install, modified replay flow with cache-preferred resolution, UI "cached" indicator in configure panel.
</objective>
