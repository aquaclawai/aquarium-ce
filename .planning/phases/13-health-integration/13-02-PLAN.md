**Part B: Add checkGatewayHealth() function**

Add a new async function `checkGatewayHealth()` before `checkConfigIntegrity()`. This polls the gateway's HTTP `/ready` endpoint for running instances:

```typescript
async function checkGatewayHealth(): Promise<void> {
  try {
    const rows = await db('instances')
      .where({ status: 'running' })
      .whereNotNull('control_endpoint');

    for (const row of rows) {
      try {
        // Derive HTTP URL from WS control_endpoint
        // "ws://localhost:19001" -> "http://localhost:19001/ready"
        const url = new URL(row.control_endpoint as string);
        url.protocol = url.protocol === 'wss:' ? 'https:' : 'http:';
        url.pathname = '/ready';

        const res = await fetch(url.toString(), {
          signal: AbortSignal.timeout(5_000),
        });

        const body = await res.json() as {
          ready: boolean;
          failing?: string[];
          uptimeMs?: number;
        };

        if (!body.ready) {
          const failingList = body.failing?.join(', ') || 'unknown';
          console.warn(`[health-monitor] gateway not ready for ${row.id} (${row.name}): failing=[${failingList}]`);

          // Broadcast degraded status to dashboard
          broadcast(row.id, {
            type: 'instance:status',
            instanceId: row.id,
            payload: {
              status: 'running',
              statusMessage: `Gateway degraded: ${failingList}`,
            },
          });
        }
      } catch {
        // Gateway HTTP unreachable -- skip (Docker status check handles container health)
      }
    }
  } catch {
    // DB query failed -- skip this cycle
  }
}
```

IMPORTANT: Do NOT create notifications on every /ready failure -- only broadcast status for real-time dashboard display. Notification spam is an anti-pattern (Pitfall 6 from research). Do NOT replace Docker status checks -- gateway checks run alongside them.

**Part C: Rewrite checkConfigIntegrity() to use gateway-authoritative hash**

Replace the entire `checkConfigIntegrity()` function body. The new implementation:

1. Query instances that are `running`, have `config_hash` set, AND have `control_endpoint` set (not just `runtime_id` -- we need the endpoint for gatewayCall).
2. For each instance, call `gatewayCall(row.id, 'config.get', {}, 10_000)` with a shorter 10s timeout (not the 30s default -- avoid blocking the slow loop).
3. Extract `result.hash` from the response.
4. Compare against `row.config_hash`. If equal, continue.
5. On mismatch: update DB `config_hash` to match gateway hash. Log the sync. Do NOT create a notification (hash drift is expected in gateway-first architecture). Do NOT call reseedConfigFiles. Do NOT insert a `config_integrity_violation` event (this is routine sync, not a violation).
6. Wrap in try/catch that skips on any error (gateway unreachable, timeout, etc.).

The new function should look like:
```typescript
async function checkConfigIntegrity(): Promise<void> {
  try {
    const rows = await db('instances')
      .where({ status: 'running' })
      .whereNotNull('config_hash')
      .whereNotNull('control_endpoint');

    for (const row of rows) {
      try {
        const result = await gatewayCall(row.id, 'config.get', {}, 10_000) as {
          hash?: string;
        };
        if (!result?.hash) continue;

        const gatewayHash = result.hash;
        const dbHash = row.config_hash as string;

        if (gatewayHash === dbHash) continue;

        // Gateway's hash is authoritative -- update DB to match
        await db('instances').where({ id: row.id }).update({
          config_hash: gatewayHash,
          updated_at: db.fn.now(),
        });
        console.log(`[health-monitor] config hash synced for ${row.id}: DB ${dbHash.slice(0, 8)}... -> gateway ${gatewayHash.slice(0, 8)}...`);
      } catch {
        // Gateway unreachable or RPC failed -- skip
      }
    }
  } catch {
    // DB query failed -- skip this cycle
  }
}
```
