<objective>
Extend template export to capture extension lifecycle state from instance_plugins/instance_skills tables, scrub OpenClaw config credentials, enforce workspace file allowlist with secret scanning and redaction, and reject local skills with scripts/assets directories.

Purpose: Template exports must fully capture the user's extension setup (plugins + skills with versions, hashes, and state) while scrubbing all credentials and preventing secret leakage through workspace files.

Output: Modified exportFromInstance (DB-only path) and reverseAdaptFromContainer (live container path) that produce templates with full extension lifecycle data, scrubbed configs, and safe workspace files.
</objective>
