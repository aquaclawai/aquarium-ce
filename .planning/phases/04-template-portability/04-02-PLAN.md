const skillsObj = (cfg.skills ?? {}) as Record<string, unknown>;
const skillEntries = (skillsObj.entries ?? {}) as Record<string, unknown>;

  const skillConfig = dbAdapter.parseJson<Record<string, unknown>>(ms.config);

  cfg.skills = skillsObj;

   b. **Create `replayPendingExtensions`** function:
   ```typescript
   export async function replayPendingExtensions(
     instanceId: string,
     controlEndpoint: string,
     authToken: string,
     userId: string,
   ): Promise<{ installed: string[]; failed: string[]; needsCredentials: string[] }>
   ```
   Implementation:
   - Call `getPendingExtensions(instanceId)` to get pending skills and plugins
   - If both arrays are empty, return early with empty results
   - For each pending skill: call `installSkill(instanceId, skill.skillId, skill.source, controlEndpoint, authToken)` wrapped in try/catch. Import `installSkill` from `./skill-store.js`. On success: if no requiredCredentials, skill is promoted to active. If has requiredCredentials, it becomes installed (needsCredentials). On failure: update status to 'failed' with error message, log warning, continue.
   - For each pending plugin: call `installPlugin(instanceId, plugin.pluginId, plugin.source, userId)` wrapped in try/catch. Import `installPlugin` from `./plugin-store.js`. On success: if no requiredCredentials, plugin needs activation (handled separately). If has requiredCredentials, it becomes installed. On failure: update status to 'failed', log warning, continue.
   - Individual failures must NOT throw -- they are logged and the loop continues
   - Return `{ installed, failed, needsCredentials }` arrays for observability
   - NOTE: installSkill and installPlugin already handle status transitions and trust checks internally. The replay just invokes them as if the user clicked install.
   - IMPORTANT: The replay should skip trust re-evaluation since these extensions were already trust-checked at template instantiation time. To achieve this, check if the extension row already has a `locked_version` set (indicating it was pre-verified during import). If so, the install functions' internal trust checks should pass since the source is preserved from import.
