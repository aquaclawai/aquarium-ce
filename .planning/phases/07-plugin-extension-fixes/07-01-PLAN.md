<objective>
Fix the platform-bridge plugin to remove RPC methods that conflict with the gateway's native handlers, fixing the root cause of the empty catalog and plugin load failures.

Purpose: The platform-bridge plugin currently registers `skills.install`, `skills.uninstall`, `plugins.install`, and `plugins.uninstall` methods that collide with the official OpenClaw gateway's native handlers. This conflict causes the plugin to fail loading entirely, which means ALL plugin-registered methods (including `skills.list`, `plugins.list`, and the built-in catalog) become unavailable -- producing an empty Available catalog after restart. Additionally, the old `plugins.install` handler wrote bad paths to gateway config, corrupting it.

Output: A clean `openclaw/plugin/index.ts` that only registers methods the gateway does NOT already have natively.
</objective>
