  <name>Task 1: Create extension-lock service (fenced mutation lock)</name>
  <files>apps/server/src/services/extension-lock.ts</files>
  <action>
Create `apps/server/src/services/extension-lock.ts` implementing the DB-only fenced mutation lock per user decision in CONTEXT.md (NOT the PRD's in-memory + DB approach).

**Design**: The lock is purely DB-backed. Acquiring = INSERT into extension_operations. The partial unique index `idx_one_active_op ON extension_operations(instance_id) WHERE completed_at IS NULL` prevents concurrent operations. If INSERT fails with unique constraint violation, return 409.

Import: `db` from `../db/index.js`, `getAdapter` from `../db/adapter.js`, `config` from `../config.js`, `randomUUID` from `node:crypto`, types from `@aquarium/shared`.

**Exported functions:**

1. `acquireLock(instanceId: string, operationType: string, targetExtension: string, extensionKind: ExtensionKind): Promise<{ fencingToken: string; operationId: string }>` 
   - Generate UUID for `id` and `fencing_token`
   - INSERT into `extension_operations` with `pending_owner = config.serverSessionId`, `completed_at = null`
   - Catch unique constraint error (SQLite error code `SQLITE_CONSTRAINT_UNIQUE` or error message containing `UNIQUE constraint failed`) → throw a typed error with `{ status: 409, activeOperation }` after querying the existing active op
   - On success return `{ fencingToken, operationId }`

2. `releaseLock(operationId: string, fencingToken: string, result: 'success' | 'failed' | 'rolled-back' | 'cancelled' | 'crashed', errorMessage?: string): Promise<void>`
   - UPDATE `extension_operations` SET `completed_at = datetime('now')`, `result`, `error_message` WHERE `id = operationId` AND `fencing_token = fencingToken` AND `completed_at IS NULL`
   - If no rows updated, log warning (stale token or already released)

3. `checkCancelRequested(operationId: string): Promise<boolean>`
   - SELECT `cancel_requested` FROM `extension_operations` WHERE `id = operationId`
   - Return boolean

4. `requestCancel(operationId: string): Promise<boolean>`
   - UPDATE `extension_operations` SET `cancel_requested = 1` WHERE `id = operationId` AND `completed_at IS NULL`
   - Return true if row updated

5. `getActiveOperation(instanceId: string): Promise<ExtensionOperation | null>`
   - SELECT from `extension_operations` WHERE `instance_id = instanceId` AND `completed_at IS NULL`
   - Return mapped to ExtensionOperation type or null

6. `cleanupOrphanedOperations(): Promise<number>`
   - UPDATE `extension_operations` SET `completed_at = datetime('now')`, `result = 'crashed'` WHERE `completed_at IS NULL` AND `pending_owner != config.serverSessionId`
   - Return count of affected rows
   - Log each orphaned operation found

**Error handling**: Create a custom `LockConflictError` class extending Error with `status: 409` and `activeOperation` payload. Export it for routes to catch.

All server .ts imports MUST use .js extension per CLAUDE.md.

<task type="auto">
  <name>Task 2: Create skill-store service (skill lifecycle CRUD with gateway RPC)</name>
  <files>apps/server/src/services/skill-store.ts</files>
  <action>
Create `apps/server/src/services/skill-store.ts` implementing skill lifecycle operations.

Import: `db` from `../db/index.js`, `getAdapter` from `../db/adapter.js`, `config` from `../config.js`, `GatewayRPCClient` from `../agent-types/openclaw/gateway-rpc.js`, `acquireLock`, `releaseLock`, `checkCancelRequested` from `./extension-lock.js`, types from `@aquarium/shared`.

Helper: Create a `mapSkillRow(row: Record<string, unknown>): InstanceSkill` function that maps snake_case DB columns to camelCase TS interface, parsing JSON columns (source, config) via `adapter.parseJson()`.

**Exported functions:**

1. `getSkillsForInstance(instanceId: string): Promise<InstanceSkill[]>`
   - SELECT * FROM instance_skills WHERE instance_id = instanceId ORDER BY installed_at DESC
   - Map rows via mapSkillRow

2. `getSkillById(instanceId: string, skillId: string): Promise<InstanceSkill | null>`
   - SELECT WHERE instance_id AND skill_id, return mapped or null

3. `installSkill(instanceId: string, skillId: string, source: SkillSource, controlEndpoint: string, authToken: string): Promise<{ skill: InstanceSkill; requiredCredentials: CredentialRequirement[] }>`
   - Acquire lock via `acquireLock(instanceId, 'install', skillId, 'skill')`
   - Wrap entire body in try/finally to always `releaseLock()`
   - Check cancel at each checkpoint via `checkCancelRequested()`
   - INSERT into instance_skills: id = adapter.generateId(), status = 'pending', pending_owner = config.serverSessionId, source = adapter.jsonValue(source), config = adapter.jsonValue({})
   - Call gateway RPC: `new GatewayRPCClient(controlEndpoint, authToken).call('skills.install', { skillId, source: source.type === 'bundled' ? 'bundled' : source.type === 'clawhub' ? source.spec : (source as any).url }, 180_000)` (3-min deadline per INFRA-07)
   - Parse RPC response for `requiredCredentials` (may be empty array)
   - If no requiredCredentials: UPDATE status → 'active', release lock with result 'success'
   - If has requiredCredentials: UPDATE status → 'installed', release lock with result 'success'
   - On RPC failure: UPDATE status → 'failed', error_message, failed_at = datetime('now'), release lock with result 'failed', rethrow error
   - Return { skill (re-fetched), requiredCredentials }

4. `enableSkill(instanceId: string, skillId: string, controlEndpoint: string, authToken: string): Promise<InstanceSkill>`
   - Acquire lock, try/finally releaseLock
   - Verify skill exists and status is 'disabled'
   - Call RPC `skills.update` with `{ skillId, enabled: true }`
   - UPDATE instance_skills SET enabled = 1, status = 'active', updated_at = datetime('now')
   - Release lock success, return updated skill

5. `disableSkill(instanceId: string, skillId: string, controlEndpoint: string, authToken: string): Promise<InstanceSkill>`
   - Acquire lock, try/finally releaseLock
   - Verify skill exists and status is 'active' or 'degraded'
   - Call RPC `skills.update` with `{ skillId, enabled: false }`
   - UPDATE instance_skills SET enabled = 0, status = 'disabled', updated_at = datetime('now')
   - Release lock success, return updated skill

6. `uninstallSkill(instanceId: string, skillId: string, controlEndpoint: string, authToken: string): Promise<void>`
   - Acquire lock, try/finally releaseLock
   - Verify skill exists
   - Call RPC `skills.uninstall` with `{ skillId }` (3-min deadline)
   - DELETE FROM instance_skills WHERE instance_id AND skill_id
   - Release lock success

7. `updateSkillStatus(instanceId: string, skillId: string, status: ExtensionStatus, errorMessage?: string): Promise<void>`
   - UPDATE instance_skills SET status, error_message, failed_at (if failed), updated_at
   - No lock needed — used internally by reconciliation

All RPC calls should use the deadline timeouts specified in INFRA-07: skills.install = 180_000ms (3 min), skills.update = 30_000ms (30s), skills.uninstall = 180_000ms (3 min).
