**Migration file** (`036_extension_tables.ts`):

Create migration using the established pattern (import `addUuidPrimary`, `addUuidColumn`, `addJsonColumn` from `../migration-helpers.js`). Create three tables per PRD sections 7.1, 7.2, 7.3:

1. `instance_plugins` table:
   - `id` (UUID primary via addUuidPrimary)
   - `instance_id` (UUID, NOT NULL, FK -> instances(id) ON DELETE CASCADE)
   - `plugin_id` (string, NOT NULL)
   - `source` (text, NOT NULL, JSON — use addJsonColumn for Postgres compat but text works for SQLite)
   - `version` (text, nullable)
   - `locked_version` (text, nullable)
   - `integrity_hash` (text, nullable)
   - `enabled` (integer, NOT NULL, default 1)
   - `config` (text/json, NOT NULL, default '{}')
   - `status` (text, NOT NULL, default 'pending', CHECK constraint for valid states)
   - `error_message` (text, nullable)
   - `failed_at` (text, nullable)
   - `pending_owner` (text, nullable)
   - `retry_count` (integer, NOT NULL, default 0)
   - `installed_at` (text, NOT NULL, default datetime('now'))
   - `updated_at` (text, NOT NULL, default datetime('now'))
   - UNIQUE(instance_id, plugin_id)

2. `instance_skills` table: Same schema as instance_plugins but with `skill_id` instead of `plugin_id`, and UNIQUE(instance_id, skill_id).

3. `extension_operations` table:
   - `id` (UUID primary)
   - `instance_id` (UUID, NOT NULL, FK -> instances(id) ON DELETE CASCADE)
   - `fencing_token` (text, NOT NULL, UNIQUE)
   - `operation_type` (text, NOT NULL)
   - `target_extension` (text, NOT NULL)
   - `extension_kind` (text, NOT NULL)
   - `pending_owner` (text, NOT NULL)
   - `cancel_requested` (integer, NOT NULL, default 0)
   - `started_at` (text, NOT NULL, default datetime('now'))
   - `completed_at` (text, nullable)
   - `result` (text, nullable)
   - `error_message` (text, nullable)
   - Index: `idx_ext_ops_instance` on (instance_id, completed_at)
   - Partial unique index: `idx_one_active_op` on (instance_id) WHERE completed_at IS NULL — use `knex.raw()` for SQLite-compatible partial unique index

For the CHECK constraints and partial unique index, use `knex.raw()` since Knex schema builder does not support SQLite partial indexes. The user decision in CONTEXT.md specifies the DB-only lock pattern relies on this partial unique index.

For timestamps: Use `text` columns with `datetime('now')` defaults (SQLite pattern, consistent with CE). Do NOT use Knex `timestamp()` helper since CE is SQLite-only.
