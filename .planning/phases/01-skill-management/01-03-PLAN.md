Replace the existing `routes/skills.ts` content with a full implementation. The existing file only has `POST /:id/skills/install` as an RPC passthrough. Replace it with routes that go through skill-store.

Import: `Router` from `express`, `requireAuth` from `../middleware/auth.js`, `getInstance` from `../services/instance-manager.js`, `getAgentType` from `../agent-types/registry.js`, all skill-store functions from `../services/skill-store.js`, `GatewayRPCClient` from `../agent-types/openclaw/gateway-rpc.js`, `LockConflictError` from `../services/extension-lock.js`, types from `@aquarium/shared`.

**Routes to implement (all behind requireAuth):**

1. `GET /:id/skills` — List managed skills + gateway built-ins (SKILL-01, UI-05)
   - Get instance (404 if not found)
   - Get managed skills from DB: `getSkillsForInstance(instanceId)`
   - If instance is running + has controlEndpoint: call `rpc.call('skills.list')` to get gateway skills
   - Separate gateway response into: managed (in DB) vs built-ins (not in DB)
   - Return `{ ok: true, data: { managed: InstanceSkill[], gatewayBuiltins: GatewayExtensionInfo[] } }`
   - If instance not running: return managed from DB only, empty gatewayBuiltins

2. `GET /:id/skills/catalog` — Browse available skills (SKILL-01)
   - Instance must be running (400 if not)
   - Call `rpc.call('skills.list')` to get all available skills from gateway
   - Map to `SkillCatalogEntry[]` format
   - Return `{ ok: true, data: SkillCatalogEntry[] }`

3. `POST /:id/skills/install` — Install a skill (replaces existing, SKILL-02, SKILL-03)
   - Instance must be running (400 if not)
   - Validate body: `{ skillId: string, source: SkillSource }`
   - Call `installSkill(instanceId, skillId, source, controlEndpoint, authToken)`
   - Catch `LockConflictError` → return 409 with activeOperation details
   - Return `{ ok: true, data: { skill, requiredCredentials } }`

4. `PUT /:id/skills/:skillId` — Toggle enable/disable (SKILL-05)
   - Instance must be running (400 if not)
   - Validate body: `{ enabled: boolean }`
   - If enabled=true: call `enableSkill(...)`, if false: call `disableSkill(...)`
   - Catch `LockConflictError` → 409
   - Return `{ ok: true, data: InstanceSkill }`

5. `DELETE /:id/skills/:skillId` — Uninstall (SKILL-06)
   - Instance must be running (400 if not)
   - Call `uninstallSkill(...)`
   - Catch `LockConflictError` → 409
   - Return `{ ok: true }`

**Error handling pattern (consistent with existing routes):**
```typescript
try { ... } catch (err: unknown) {
  if (err instanceof LockConflictError) {
    res.status(409).json({ ok: false, error: err.message, activeOperation: err.activeOperation });
    return;
  }
  const message = err instanceof Error ? err.message : String(err);
  res.status(500).json({ ok: false, error: message });
}
```

All routes use `req.auth!.userId` for getInstance, and extract `controlEndpoint`/`authToken` from the instance object.

**extension-credentials.ts**: Create new route file for extension-scoped credential operations (SKILL-04). This is separate from the existing `credentials.ts` per user decision in CONTEXT.md.

Import: `Router` from `express`, `requireAuth` from `../middleware/auth.js`, `getInstance` from `../services/instance-manager.js`, `addCredential` from `../services/credential-store.js`, `acquireLock`, `releaseLock` from `../services/extension-lock.js`, `LockConflictError` from `../services/extension-lock.js`, `GatewayRPCClient` from `../agent-types/openclaw/gateway-rpc.js`, types from `@aquarium/shared`.

Route:

`POST /:id/extension-credentials` — Save extension-scoped credential + inject via config.patch
   - requireAuth
   - Get instance (404 if not found), must be running (400)
   - Validate body: `{ provider: string, credentialType: string, value: string, extensionKind: ExtensionKind, extensionId: string, targetField: string }`
   - Acquire extension lock: `acquireLock(instanceId, 'configure', extensionId, extensionKind)`
   - try/finally releaseLock
   - Store credential via `addCredential(instanceId, provider, credentialType, value, { extensionKind, extensionId, targetField })` — the metadata object binds the credential to this extension
   - Call `rpc.call('config.patch', { ... })` with 30s timeout (INFRA-07) to inject the credential into the extension's scoped namespace. The config.patch payload should set the credential in the extension's config namespace: for skills, `skills.entries.<extensionId>.env.<targetField>` or `skills.entries.<extensionId>.apiKey`; for plugins, `plugins.entries.<extensionId>.config.<targetField>`. Use SecretRef format: `{ source: 'env', id: 'AQUARIUM_CRED_<hash>' }` where hash is derived from `${extensionKind}_${extensionId}_${targetField}`.
   - On config.patch success: if extensionKind is 'skill', UPDATE instance_skills SET status = 'active', updated_at WHERE instance_id AND skill_id AND status = 'installed'. (Promoting from installed to active after credential configured.)
   - On config.patch failure: leave status as 'installed', log warning, return partial success response
   - Release lock
   - Return `{ ok: true, data: { credentialStored: true, configPatched: boolean } }`
   - Catch LockConflictError → 409
