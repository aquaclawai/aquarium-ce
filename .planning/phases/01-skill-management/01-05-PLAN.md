Create `apps/web/src/components/extensions/` directory with the following files:

**ExtensionsTab.tsx** — Main container component.

Props: `{ instanceId: string; instanceStatus: string }` (matches existing tab component pattern).

State:
- `subTab: 'plugins' | 'skills'` — default 'skills' (plugins are Phase 2, show placeholder)
- `managedSkills: InstanceSkill[]`, `gatewayBuiltins: GatewayExtensionInfo[]`
- `catalog: SkillCatalogEntry[]`
- `loading: boolean`, `error: string | null`
- `installing: string | null` (skillId currently being installed)
- `configuringSkillId: string | null` (skillId whose gear icon was clicked — used by CredentialConfigPanel in Plan 01-06)

Effects:
- On mount and when subTab changes: fetch data
  - Call `api.get<{ managed, gatewayBuiltins }>(`/instances/${instanceId}/skills`)` always (works when stopped — returns DB data)
  - Call `api.get<SkillCatalogEntry[]>(`/instances/${instanceId}/skills/catalog`)` only if instanceStatus === 'running'
- Add a manual refresh icon button (circular arrow icon via CSS/unicode) that re-fetches

Layout (per user decisions in CONTEXT.md):
```
[Plugins pill] [Skills pill]          [refresh icon]

--- Installed ---
<SkillRow /> for each managedSkill
(or "No skills installed" empty state)

--- Gateway Built-ins ---    (read-only section, UI-05)
<read-only rows for gatewayBuiltins — show name, version, enabled status, no actions>

--- Available ---
<CatalogSkillRow /> for each catalog entry not already installed
(or "Start instance to browse available extensions" message when stopped)
(or skeleton loading cards while fetching)
```

Sub-tab toggle: Rendered as pill/segmented control buttons per user decision. Use CSS classes `.sub-tab-toggle`, `.sub-tab-toggle button`, `.sub-tab-toggle button.active`.

When `instanceStatus !== 'running'`: Show installed section from DB. Hide catalog section, show message `t('extensions.catalog.startInstance')`. Disable all mutation buttons (install, toggle, uninstall, gear).

Skeleton loading: Show 3 skeleton row placeholders (gray animated bars) while loading. Use CSS `@keyframes skeleton-pulse` animation.

Action handlers:
- `handleInstall(skillId, source)`: POST to install endpoint, refresh list on success
- `handleToggle(skillId, enabled)`: PUT to toggle endpoint, refresh on success  
- `handleUninstall(skillId)`: DELETE to uninstall endpoint, refresh on success
- `handleConfigure(skillId)`: Sets `configuringSkillId` state to the given skillId. Does NOT open any modal or call any API — the CredentialConfigPanel that reads this state is built in Plan 01-06.
- All mutation handlers: set `installing` state for loading indicator, catch errors and set error state

Use `useTranslation()` for all strings (namespace: `extensions.*`).

**SkillRow.tsx** — Single installed skill row.

Props: `{ skill: InstanceSkill; onToggle: (skillId, enabled) => void; onUninstall: (skillId) => void; onConfigure: (skillId) => void; disabled: boolean; }` 

Render per user decision:
- Row layout: `icon | name | description (truncated to ~60 chars) | colored dot + status text | toggle switch | gear icon | uninstall X`
- Status dot colors: green=active, yellow=installed/degraded, red=failed, gray=disabled
- Toggle: `<input type="checkbox" checked={skill.enabled} onChange={...} disabled={disabled || skill.status === 'failed'} />`
- Gear icon: button with settings unicode char (e.g. \u2699). onClick calls `onConfigure(skill.skillId)`. This ONLY sets `configuringSkillId` state in the parent ExtensionsTab — no modal, no window.prompt(), no API call. The CredentialConfigPanel that renders when `configuringSkillId` is set will be built in Plan 01-06.
- Uninstall X: button with X char, onClick calls onUninstall with confirm dialog

**CatalogSkillRow.tsx** — Single available catalog skill row.

Props: `{ entry: SkillCatalogEntry; onInstall: (skillId, source) => void; installing: boolean; disabled: boolean; }`

Render per user decision:
- Row layout: `icon | name | description | "Bundled" source badge | key icon + "Requires API key" if entry.requiredCredentials.length > 0 | Install button`
- Source badge: green pill with text "Bundled" or "ClawHub"
- Install button: `<button onClick={...} disabled={disabled || installing}>Install</button>`
- Key icon: lock/key unicode char

**ExtensionsTab.css** — All styles.

Use CSS variables from index.css (Oxide design system). No Tailwind per CLAUDE.md.
- `.extensions-tab` container with padding
- `.sub-tab-toggle` flex row with gap, buttons styled as pills (border-radius, background toggle)
- `.section-header` styled like existing section headers
- `.skill-row` flex row with gap, aligned items, border-bottom for separation
- `.status-dot` small circle (8px) with status colors using CSS variables
- `.source-badge` small pill with background color
- `.skeleton-row` with `@keyframes skeleton-pulse` animation
- `.gateway-builtins` section with slightly dimmed styling to indicate read-only
- Responsive: rows stack on small screens

All colors use `var(--color-*)`. No hardcoded hex values.
